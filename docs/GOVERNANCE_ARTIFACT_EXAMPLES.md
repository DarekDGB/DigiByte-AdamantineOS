# Governance Artifact Examples

This document shows real example artifacts produced during the first
governance handshake between **Adaptive Core v3** and **Adamantine
Wallet OS**.

Date of demonstration: **2026‑02‑26**

These artifacts prove the deterministic governance pipeline implemented
between the two systems.

------------------------------------------------------------------------

# 1. Upgrade Proposal Artifact

Generated by **Adaptive Core v3**.

Purpose: Adaptive Core detects a condition and proposes a configuration
or system change.

Example:

``` json
{
  "v": "upgrade_proposal_v3",
  "proposal_id": "AC-DEMO-0001",
  "proposal_hash": "0aff6779096c20d90a06f31d30aed1896ce59faf86e0fb0b1492072a976be416",
  "domain": "SECURITY_THRESHOLDS",
  "summary": "Detected drift_score 0.8000 >= threshold 0.5000.",
  "action": "INCREASE_THRESHOLD",
  "target": {
    "component": "eqc_engine",
    "version": "3.1.0"
  },
  "guardrails": ["AMG-001"]
}
```

Key properties:

-   **proposal_id** --- unique proposal identifier\
-   **proposal_hash** --- deterministic canonical hash of proposal
    content\
-   **domain** --- subsystem affected by the change\
-   **action** --- recommended change\
-   **target** --- component affected\
-   **guardrails** --- governance rules referenced

------------------------------------------------------------------------

# 2. Review Receipt Artifact

Generated by **AdamantineOS** after validating the proposal.

Purpose: Provide an auditable record of the governance decision.

Example:

``` json
{
  "v": "ac_review_receipt_v1",
  "proposal_id": "AC-DEMO-0001",
  "proposal_hash": "0aff6779096c20d90a06f31d30aed1896ce59faf86e0fb0b1492072a976be416",
  "decision": "APPROVE",
  "receipt_hash": "99f272c75119207fb3b83eea176aff08af72f506472322dc9e717dfe2c4a675d",
  "reviewer_id": "reviewer:demo",
  "reviewed_utc": "2026-02-26T00:00:00Z"
}
```

Key properties:

-   **decision** --- APPROVE or DENY\
-   **receipt_hash** --- deterministic record hash\
-   **reviewer_id** --- decision authority\
-   **reviewed_utc** --- timestamp of evaluation

------------------------------------------------------------------------

# 3. Enforcement Decision Artifact

Produced by AdamantineOS enforcement logic.

Purpose: Convert the review receipt into a deterministic **allow /
deny** system decision.

Example:

``` json
{
  "allow": true,
  "proposal_hash": "0aff6779096c20d90a06f31d30aed1896ce59faf86e0fb0b1492072a976be416",
  "receipt_hash": "99f272c75119207fb3b83eea176aff08af72f506472322dc9e717dfe2c4a675d",
  "reason_id": "REVIEW_RECEIPT_APPROVE"
}
```

Possible outcomes:

-   **allow: true** → proposal accepted
-   **allow: false** → proposal rejected

# Governance Pipeline

The full deterministic governance flow:

Adaptive Core\
→ generates `upgrade_proposal_v3`

AdamantineOS\
→ validates artifact\
→ recomputes canonical hash\
→ evaluates policy

AdamantineOS\
→ produces `review_receipt_v1`

AdamantineOS\
→ outputs final allow/deny decision
