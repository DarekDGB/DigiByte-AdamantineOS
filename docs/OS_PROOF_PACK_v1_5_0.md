## 1. Purpose

The OS Proof Pack exists to prove, in CI, that Adamantine evaluation is:

- deterministic
- fail-closed
- contract-stable
- regression-locked

v1.5.0 extends the sealed foundation by **freezing the mobile integration contract** so external teams can integrate without guessing. fileciteturn0file0

---

## 2. What v1.5.0 Adds (Contract-Level)

- Mobile request contract **v2** (normative)
- Mobile response contract **v2** (normative), including `protection_mode`
- Strict JSON schemas (reject unknown fields)
- Golden fixture roundtrip pack + hash manifest
- Reason registry for mobile UI mapping + hash lock
- Conformance suite that fails on any payload shape drift

---

## 4. Determinism Requirements

For each fixture, repeated runs MUST produce identical:
- `decision`
- `protection_mode`
- `reason_id`
- `context_hash`
- artifacts shape

CI MUST execute multi-run determinism (recommended: **100 repeats** per fixture).

---

## 5. Fail-Closed Requirements

CI MUST prove:
- unknown request field → deterministic deny (or schema reject before evaluation)
- missing required request field → schema reject
- wrong type → schema reject
- unknown response field (generated by code drift) → schema reject
- reason_id must exist in mobile reason registry (or CI fails)
