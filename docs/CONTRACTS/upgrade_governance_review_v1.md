# Governance Review Contract

## Overview

This document describes the contract governing review of upgrade
proposals between Adaptive Core and AdamantineOS.

Two artifact types are involved:

1.  `upgrade_proposal_v3`
2.  `proposal_review_receipt_v1`

------------------------------------------------------------------------

# upgrade_proposal_v3

Generated by Adaptive Core.

Contains:

-   proposal metadata
-   version target
-   deterministic canonical representation
-   proposal hash

Requirements:

-   Must validate against schema
-   Must produce deterministic canonical JSON
-   Hash must match canonical representation

------------------------------------------------------------------------

# proposal_review_receipt_v1

Generated by AdamantineOS after evaluation.

Contains:

-   proposal_hash
-   decision
-   review timestamp
-   policy result

Decision values:

-   APPROVE
-   DENY

------------------------------------------------------------------------

# Verification Process

AdamantineOS performs the following checks:

1.  Validate proposal schema.
2.  Canonicalize JSON representation.
3.  Recompute proposal hash.
4.  Compare with provided hash.
5.  Evaluate governance policy.

If any step fails -> proposal is rejected.

------------------------------------------------------------------------

# Security Guarantees

The review process ensures:

-   deterministic governance decisions
-   verifiable audit trail
-   tamper detection via hashing
-   cross-repository compatibility

------------------------------------------------------------------------

# Contract Stability

Changes to the proposal format require:

-   schema version bump
-   CI compatibility updates
-   explicit documentation update
